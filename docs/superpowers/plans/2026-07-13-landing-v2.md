# robium.org landing v2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reposition the landing page from "Claude Code plugin" to "the Physical AI expertise AI agents need" — new hero with CSS-only agent install tabs, a curated skill grid grouped by the subtitle's four pillars, five how-it-works steps, and no hardcoded numbers anywhere.

**Architecture:** Astro 6 static site, zero client-side JS on the landing page. All changes are `.astro` components plus one new data file. The install tabs are CSS-only (hidden radio inputs + `:checked ~` sibling selectors) specifically so the zero-JS guarantee survives. `tests/smoke.sh` pins literal page strings, so **every task updates its own smoke assertion in the same commit** — that is the test cycle.

**Tech Stack:** Astro 6, plain CSS (scoped per-component), bash smoke test.

**Spec:** `docs/superpowers/specs/2026-07-13-landing-v2-design.md`

## Global Constraints

- **No hardcoded numbers on the page.** No skill counts, no star counts. Applies to copy AND to generated markup (`SkillsGrid.astro:7` currently renders `{skills.length}` — that is a violation to remove).
- **Zero client-side JS on the landing page.** No React island, no `<script>`. Tabs must be CSS-only.
- **Aspirational copy is allowed; runnable-but-broken commands are not.** The Cursor and Gemini tabs must NOT contain a command a visitor can copy and run, because those manifests do not exist yet. They show state + the example ask instead.
- **Card descriptions are never hand-written.** They come from `src/data/skills.json`, generated at build time by `scripts/fetch-skills.mjs`.
- **The done bar is `make smoke`.** It must print `SMOKE PASS` at the end of every task.
- Dev loop: `npm run dev:site -- --port 4322` (site only — port 4321/8080 belong to another agent; do NOT run `npm run dev`).
